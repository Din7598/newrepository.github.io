First, LAN is subset in a network, quick reminder a network is collection of connected nodes via a media be it wired or wireless.

A node can be any device with the ability to send and receive data, example is a phone or printer.

LAN simply put stand for Local Area Network. In LAN connection the focus lays main on 2/3 devices, I say this because one of this devices is still maybe in use by people, but I don't understand why? they'd even be using this device especially with the rate cyberwarfare on a full blown rise. These three devices are a switch, a router and a hub.

Now let's dive deeper starting with a switch and the role it plays in LAN.

What is a switch?

A switch is a network device, which end devices primarily connect to keep in touch with the network. A switch has a MAC address and also a lot of ports where you plug in devices to connect them to it such as a router, another switch and even end devices such as PC or printer. Each port on a switch is assigned a MAC address. A MAC address full disclosure means "Media Access Control", it is a unique identification number assigned to network interface card or its also a physical address. The address itself is 48 bits or 6 bytes long. The address has two sections, the first section consist of the first 3 bytes is called the "organisationally unique identifier" used in identifying the vendor. The second is a unique value assigned by the vendor known as the "unique identifier".
Earlier, I made a statement "2/3 devices" now the shorthanded device amongs the 3 devices, is not the switch but rather its called a hub. A switch operates at layer 2 of the OSI model datalink. A switch has a single broadcast domain at first, in the sense that when its plugged into a network the first time any frame it receives it just forwards them out of each and every port where it feels a device is connected to except the port where it initially received the frame. Once this has happened the switch creates what is called a MAC address table

A broadcast domain is a range of devices that can receive a broadcast frame sent from a network device. In a broadcast domain the end devices have to be connected to a network device such as a switch. However, In a switch there's still the issue of broadcast storms happen. "What is a broadcast storm? you might ask, simply put it is when a message meant for a particular individual end devices ends up going out to that individual end device and other devices. This will occassionally if switches are connected eachother but with the invention of Spanning Tree Protocol it rarely does happen. A switch creates a collision domain for each node connected that is connected to it. In other to prevent collisions from happening in a switch, the CSMA/CD (Carrier Sense Multiple Access with Collision Detection) has been introduced in switches to prevent collisions from happening. A switch creates these LANs as devices connected to a switch form a LAN.

How does it work?

Befroe a switch sends out a frame it listens to where another frame is being transported to and from the network. And if by chance a collision does occur a jamming signal is sent to prevent any further collisions from happening.

What is a hub?

I mentioned a hub earlier, a hub is a device similar to a switch but it is more of dull device.

Its said to be dull in the sense that hub have a single collision and broadcast domain, so aside from a broadcast storm happening expect a collision. A collision is when nodes attmept to transmit frame both send the frame at the same time, when this frames cross path on the ethernet then a collision is sure to happen.

What is a router?

A router is a network device, that is responsible for creating a path for packets to flow in a network using routing protocol across a separate LANs. A router operates in layer 3 of OSI model. Router links together different LAN. Routers by default separate broadcast domains.

Functionalities a router is responsible for in a LAN:

  1. Packet switching
  2. Packet filtering
  3. Route selecting, using it routing table.
  4. internetworking between LANs.

Types of transmission:

  1. Broadcast : From a device to all devices connected to it.
  2. Multicast : From a device to a selected few devices connected.
  3. Anycast : From a device to a singular device connected.

Since this is about LAN it would also make sense to discuss the things that cause LAN traffic:

  1. Too many devices in the broadcast domain.
  2. Broadcast storms.
  3. Low bandwidth.
  4. Muliticasting

All of these devices put together make up the internet.

What is the internet?

Simply put the internet is a large scale computer network made up of a lot of routers, switches and nodes connected together.

What makes a good network?
